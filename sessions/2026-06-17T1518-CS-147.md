# CS-147 — Sentry-fix loop proven end-to-end (supervised) + CS-50 CI source-copy retirement

## Summary

Two ships this session. **CS-147 (primary)** — the Sentry-fix auto-remediation loop (Phase 3 of the CS-375 umbrella) ran its first full cycle end-to-end under supervision and the independent evaluator proved **discriminating** on real data. The loop-harness tree, previously untracked on `main` (a reproducibility risk before any unattended run), is now committed, with two hardening fixes found live during the cycle. **CS-50 (secondary)** — retired the stale agent-system source copy of `weekly-security.yml`, which had become a false source-of-truth and caused a silent drift; each product repo's copy is now canonical.

Both commits were already on `main` at handoff time (15:00 UTC); this entry is the durable record. **Neither ticket is Done** — see What's next.

## What was done

### CS-147 — Sentry-fix loop, first supervised cycle (commit `01892ba917`)

- **Cycle 1 target:** `NODE-API-PROD-C` — the `POST /api/apps` Drizzle `ilike` crash. Ran the full loop: **filter → Graphify trace → fix → independent evaluator vs real data → PR → STOP.**
- **Evaluator is discriminating (supervised-first goal met):**
  - Known-good fix (`ilike` → `lower(name) = lower(input)`) → **PASS**.
  - A deliberately-broken variant (one `lower()` dropped) → **NEEDS_WORK** — caught via `vgera_ro` real-data **reproduction**, not by diff-reading. No `evaluator.md` divergence between the two runs.
- **Shipped:** api_node **PR #278** → `development`, evaluator-PASS, reviewer @jthakkar1981. **Open, awaiting human merge.**
- **Harness now tracked on agent-system `main`** (was untracked — reproducibility risk). 10 files / +382 under `loop-harness/`: `commit-on-stop.sh`, `kill-switch.sh`, `steer.sh`, and `sentry-fix/{README.md, RUNBOOK.md, builder-prompt.md, evaluator.md, ingest-rest.sh, issue-filter.md, routing.md}`.
- **Two hardening fixes found during the supervised cycle (both hard blockers before unattended):**
  1. `commit-on-stop.sh` — the **CS-404 untracked-files bug recurred live**: `git commit -am` plus the untracked sweep checkpointed `node_modules` symlinks and `.claude/` harness files onto the work branch. Now stages **both** tracked and untracked changes **by explicit path**, drops `node_modules` / `.claude/` / symlinks / conflict-copies / sentinels via `should_skip()`, and commits with `-m` only (never `-am`).
  2. `sentry-fix/README.md` — hook-wiring snippet now uses `$CLAUDE_PROJECT_DIR` absolute paths; relative `.claude/hooks/…` does not fire when CC runs hooks from a subdir.
- Supervised-first proof result recorded in `sentry-fix/RUNBOOK.md` (good fix → PASS, subtly-broken fix → NEEDS_WORK, both via `vgera_ro` real data) and logged to the CS-147 Linear ticket.

### CS-50 — retire stale `weekly-security.yml` source copy (commit `641da0a6fd`)

- The agent-system source copy at `github-actions/weekly-security.yml` had **diverged from all three live per-repo copies** (web-angular / api_node / Colobbo-Mobile) and was synced to none — a false source-of-truth that caused a silent drift (a Node-24 action bump that never reached any product repo).
- `weekly-security` is **per-repo customised** (api_node uses a matrix; each repo has its own name/Slack target), so it is not a shared template and belongs in each product repo, owned and reviewed in-repo.
- Each product repo's `.github/workflows/weekly-security.yml` is now canonical. `wiki/raku-meta/graphify-coverage-audit.md` updated with a retirement pointer.
- `ai-review.yml` and `slack-user-map.json` remain in `github-actions/` (genuinely shared; assessed separately).
