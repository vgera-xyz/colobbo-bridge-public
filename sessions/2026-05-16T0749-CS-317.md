# CS-317 — sync-bridge-public.sh durability hardening

## Summary

Session opened on a `/cibo-start-session` anomaly: an uncommitted line in `memory/incoming/sync-failures.md` (`2026-05-16T00:59:55Z — git pull --ff-only failed on mirror`, written by the prior CS-311 handoff). Diagnosed it to root cause, applied an immediate unblock, then shipped a durable fix under full `/plan → /rthink Pass 1 → execute → verify` methodology. **Not a branch divergence** — the public bridge mirror remote (`vgera-xyz/colobbo-bridge-public`) was healthy throughout (`main = b00328f5`). The script's persistent local clone in `/tmp/colobbo-bridge-public-init` was **gutted by the macOS /tmp reaper** (`.git/HEAD` + `.git/config` + `.git/description` deleted; `refs/`, `logs/`, `objects/` survived) → git no longer recognised it as a repo → the existence-only guard `[ -d "$PUBLIC_REPO_DIR/.git" ]` false-passed and re-failed every handoff forever. Shipped `3d83b7dc` on main (CS-317 → Done).

## Diagnosis (session-start → root cause)

- Empirically probed `/tmp/colobbo-bridge-public-init`: `.git/` present but `HEAD`/`config`/`description` MISSING; `refs/heads/main` + `refs/remotes/origin/main` + `logs/HEAD` + `objects/` intact. `git -C` → "not a git repository" (exit 128).
- `git ls-remote` on the public mirror → `main = b00328f54dbfe41fa548147276ffdadcc0e60e55`, current through last good sync `2026-05-15T13:17:27Z`. Remote healthy; only the local scratch clone damaged.
- Self-perpetuating: the `-d .git` guard sees the gutted dir, skips the re-clone path, runs `git pull` on a non-repo → fails → logs → exits 0 (ADR-017 contract) → repeats next handoff. Mirror stale since 2026-05-15T13:17Z (missing the CS-311 ship entry); zero data risk (one-way mirror, private = source of truth).

## Immediate unblock (applied this session)

`rm -rf` blocked by `block-dangerous-commands.py` hook → moved the corrupt clone aside instead: `mv /tmp/colobbo-bridge-public-init /tmp/colobbo-bridge-public-init.corrupt.20260516T013541Z`. The `-d .git` guard now sees no dir → next `/cibo-handoff` re-clones fresh from the healthy remote and auto-catches-up CS-311 + this entry. Reversible; `/tmp` self-reaps the corpse.

## Methodology — /plan → /rthink Pass 1 → execute

`/rthink` Pass 1 on the 4-task plan found **2 HIGH + 4 MED** gaps *before any code was written*:

- **HIGH-1 — predicate ascent.** Moving the clone *inside* this repo means a naive `git -C <dir> rev-parse --is-inside-work-tree` ascends to `colobbo-agent-system/.git` and false-passes on an empty/gutted scratch dir. Fixed: predicate asserts `rev-parse --show-toplevel` **equals the dir's own `pwd -P`** (defeats ascent) AND `remote get-url origin == $PUBLIC_REPO_URL`.
- **HIGH-2 — bash `local` masking.** `local x=$(cmd) || return 1` never fires (the `local` builtin's exit status masks the substitution's). Fixed: declaration split from assignment.
- MED: rm-path catastrophic guard, clone-dir → `scripts/.bridge-mirror/`, self-heal observability breadcrumb, 3 verify-before-build items (all resolved clean: cibo-handoff invokes the script no-env from repo root; no pre-existing `.scratch`/`.bridge-mirror`; canonical_pages keys ADR-017 by path only).
- A **third** gap surfaced during test-harness *design* (not code): the GAP-3 rm guard was first written as a name allowlist (`*/.bridge-mirror|*/colobbo-bridge-public-init`) which would break the legitimate `BRIDGE_MIRROR_DIR` override. Re-shaped to a catastrophic-target guard (`""|/|.|..|$PWD|$HOME`).

## Learnings (methodology — future cycles inherit)

1. **Session-start anomalies are root-cause work, not noise.** A 1-line uncommitted operational-log entry surfaced a self-perpetuating outage class; treating it as "just commit it" would have masked a recurring silent failure.
2. **Moving scratch state *into* a git repo changes the validity-probe contract.** Any `git -C <subdir> rev-parse` ascends to the parent repo — existence/`-d .git` checks and `--is-inside-work-tree` both mislead. Use `--show-toplevel == pwd -P(self)` for "is this dir its OWN repo root".
3. **`local x=$(cmd) || handler` is dead code.** Split declaration from assignment whenever the substitution's exit status matters (`bash-jq-reflexes` class).
4. **Self-healing without observability just moves the silence.** A re-clone that emits nothing means a recurring corruption is never noticed. The stderr breadcrumb is the difference between "fixed" and "fixed and known".
