## Key clarification captured (for the next session / the runbooks)

Sentry is three separable jobs: (1) **crash reporting** — runtime via DSN, always on, deploy-independent; (2) **sourcemap upload** — web/mobile `build:prod` does it, backend does NOT (its `build` is plain `tsc`; upload lives only in `sentry-release.yml` CI); (3) **release ceremony** (create/set-commits/finalize/deploy → "fixed in vX", regression flags, deploy timeline) — **no build script does this**; only the CI workflows (api_node auto on push-to-main once #383 lands; web #633 + mobile #381 dormant). So on the **EC2 backend deploy, Jignesh runs NO Sentry commands** — CI owns it; the only EC2-side nicety is setting `SENTRY_RELEASE` in the app env so events tag to the CI-created release.
