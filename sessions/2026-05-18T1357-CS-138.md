### 1. ARCHITECTURE (what Raku ingests at extraction)

- **Three-layer substrate model:** Layer 0 = `lobes/_shared/<id>/<version>.md` (cross-cutting prose, ≥2 consumers); Layer 1 = `lobes/<lobe>/LOBE.md` (Lobe-unique methodology); Layer 2 = `glaze/<ware>.glaze.json` (per-Ware config). Orthogonal to ADR-013's memory layers (terminology table in ADR-032 §1 / lobe-substrate-layers.md).
- **Inherit-by-default / override-by-exception** — 4 patterns: `includes:` (+params) / `overrides:` (full body) / `extensions:` (additive) / omit (Pattern 4 — local prose or concern N/A). Per-id mutual exclusivity (Gate 19).
- **Parameterised fragments (Option B):** fragment body carries `{{param.NAME}}`; consumer supplies values inline in the `includes:` block. Splice = replace single-line `<!-- INCLUDE: id@ver -->` marker (whole line incl `\n`) with fragment-body (frontmatter stripped, one trailing `\n` trimmed) after `{{param.*}}` expansion.
- **Versioned path-as-truth:** `id@version` → `_shared/<id>/<version>.md` directly (no registry lookup). Cargo/Maven/Helm pattern. Breaking change = add `<id>/2.0.0.md`; v1 consumers untouched.
- **Compound versioning:** core semver (manual, own methodology) + per-include `id@version#paramhash` + `composite_signature` (registry-only — placing it in deployed-agent frontmatter would break byte-identity). Param/fragment change moves signature, not core semver; ratified by system VERSION tag.
- **Two-pass compose engine:** pass 1 splice+`{{param.*}}` (both deploy + portability modes); pass 2 existing `{{glaze.*}}`. No-`includes:` LOBE.md ⇒ byte-identical to pre-Layer-0 (backcompat invariant, Gate-14-proven).
- **Lobe Registry:** `lobes/_registry.json` filesystem-derived (`compose.sh registry-regen`), deterministic JSON, Cargo.lock pattern; Gate 19 `--check` = "lockfile up to date".
- **Gate 19:** registry integrity (on-disk==regen; ref resolution; param-schema; per-id exclusivity; `_shared` frontmatter depends_on:[] + path-is-truth). NOT Gate 15 (taken).
- **`_compose_version` frozen at 2.0.0** — Checkpoint-1 7f ratification (this chat is the authority, recorded in ADR-032 §7): the splice engine is strictly additive (`{{param.*}}` disjoint from `{{glaze.*}}`); the token-resolution contract is unchanged; engine-capability provenance moved to `registry.schema_version`. DEFAULT #6 read operationally as "byte-identical to v2.32.2 form".
- **DNA Principle codified:** same architecture, unique intelligence per user via Glaze (Layer 2) + Memory — encoded in the substrate mechanically.

### 2. PRINCIPLES (durable)

- **Layer 0 inherit-by-default / override-by-exception.** memory-lobe Pattern-4-omit is the **first production demonstration**: until this ship Pattern 4 was theoretical; Memory's empirical applicability scan (0 status-keyed terminals; soft-warnings/advisories model) is the validating evidence. The single-sibling safety belt itself validating substrate design is the meta-lesson.
- **Bulk-edit-not-Lobe-by-Lobe.** System-wide change = one `_shared/` edit + `compose deploy --all` + ONE tag covers the fleet. Empirically demonstrated across 11 Lobes via tooling (CC = tool-operator, not hand-editor).
- **System-version-as-truth.** Repo `VERSION` authoritative; registry `system_version` tracks it (co-regen-or-Gate-19-drifts coupling).
- **Substrate-is-means-not-end.** Built for production correctness (byte-identity keystone, gates), not perfection; Pass-2 HALT + fallback were real control points.
- **State-drift verification + multi-blocker wrong-target rule.** Verify-by-grep against authoritative source; ≥2 independent structural blockers = wrong-target signal (carried from the prior CS-138 build-discovery bridge).
