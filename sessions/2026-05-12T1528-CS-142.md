## Architectural decisions (codified in ADR-020)

1. **4-upstream multi-pin pattern.** First Lobe to exercise ADR-007's "future N-upstream Lobes contribute N entries — same shape, no further ADR" generalisation at N=4. The pattern extends cleanly to N=5 when CS-142 v0.1.2 adds MobileResult.

2. **Spec-level-only ingestion (Option B rejected).** The Lobe ingests upstream Lobe JSON outputs, NOT source code. By pipeline position, no code exists at invocation time (Security runs BEFORE code-writer). Reading source would duplicate security-auditor's substrate, breaking CS-22 non-duplication AND violating ADR-005 layering.

3. **Petri-NOT-AppSec category-error rationale.** Per [Linear doc 033ec31de572](https://linear.app/colobbo/document/033ec31de572) deep critical assessment: Petri is an LLM alignment auditor (38 dimensions: deception, sycophancy, power-seeking) NOT an AppSec tool. CS-293 v0.2.0 re-scoped to OWASP AISVS + Top 10 for Agentic Applications + AIVSS as primary fixtures; Petri remains as one optional probe.

4. **Fixture vocabulary disambiguation.** `lobes/security-lobe/fixtures/` = rule-fixtures (detection corpus); `lobes/security-lobe/test-fixtures/` = regression-test-fixtures (Gate-18 harness inputs). Both ship.

5. **Top 10:2025 RC stance.** Ship against the Release Candidate (not stable 2021). Machine-readable in `glaze.security.rc_framework_stance.top10-2025-mapping`; `framework_rc_stance` advisory emitted on every run. SSRF-folded-into-A01 captured as explicit stub.

## Schema fix during authoring (mid-execution)

The initial `fixture_id` pattern in both schemas (`^[a-z0-9-]+/[a-z0-9-]+$`) didn't allow `.` in the framework portion, but `framework` field's pattern (`^[a-z0-9.-]+$`) DID allow it. Inconsistency surfaced when 30 ASVS fixtures all failed validation on first run (framework: `asvs-5.0-l2` contains `.`). Fixed in C3 by relaxing both `fixture_id` patterns to `^[a-z0-9.-]+/[a-z0-9.-]+$`. Schema versions unchanged — pattern relaxation is a PATCH-level fix per docs-only convention; `_metadata.current_version` stays at 1.0.0.

## CS-22 non-duplication evidence (mechanically enforced)

Every asvs-5.0-l2 rule-fixture declares:
- `not_covered_by[]` — non-empty array of existing security tools that DO NOT cover this rule. Validator-enforced (validate-security-fixture.mjs special-cases asvs-5.0-l2 to reject empty arrays).
- `not_covered_by_rationale` — ≥ 50 chars (schema-enforced minLength) explaining WHY each listed tool doesn't cover this rule. Discipline: state the tool's input substrate (source code, manifest, file content) and explain why our substrate (spec-level upstream JSON) differs.

### Discipline lesson

The Lobe Regression Suite is `workflow_dispatch` only — does NOT auto-run on push. Local pre-push gates do NOT exercise per-Lobe regression harnesses. Drift accumulates silently until manually dispatched. The April-May 2026 ship cadence (CS-139 → CS-140 → CS-141 → CS-281 → CS-142) had no automatic mechanism enforcing the well-documented "every fintech glaze edit pairs with a sibling fixture re-snap" pattern. CS-297 closes this gap structurally before CS-143 starts.
