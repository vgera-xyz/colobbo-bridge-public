## What was built

Seventh Raku Lobe — **Mobile Lobe v0.1.0** — shipped as the second 2-upstream consumer per ADR-007 §Generalisation (after Database). Cross-platform mobile framework specialist consuming ArchResult + BackendResult, emitting MobileResult with per-component-per-platform implementation specs (navigation routes, plugin wrappers, sync-queue boundary, secure-storage bindings, offline-first persistence) for downstream code-writer + test-writer sub-agents.

Bundled with **ADR-019 auth-lift** — promoting `glaze.backend.auth.*` to top-level `glaze.auth.*` per ADR-006's ≥2-current-consumer rule (Backend = consumer #1 since CS-138; Mobile = consumer #2 at this ship). Three-commit atomic sequence (additive → consumer migration → cleanup) per ADR-019 §Decision 2.

### Phase 2 path-discipline cleanup (Pattern E surfaced)

Phase 1 LOBE.md authoring had 14+ forbidden-term leaks (Capacitor, Ionic, Angular, Colobbo, MSSQL, etc. in prose AND in `glaze.mobile.supported_component_types` enum values like `capacitor-page`, `ionic-component`). Phase 2 verification (`compose.sh mobile-lobe glaze/colobbo.glaze.json` + grep against `.claude/rules/three-layer.md` extended-regex) exposed 18 leaks in colobbo-compiled output. Surgical 14-Edit refactor of LOBE.md prose + agent.json description + generalisation of `glaze.mobile.supported_component_types` to framework-neutral values (`mobile-page`, `mobile-ui-component`, `plugin-wrapper`, etc.). Final state: 0 leaks in BOTH colobbo-compiled and fintech-compiled outputs.

Pattern E (filesystem-path errors in chat-authored briefs) AND the new vector of "framework-prefixed values in glaze that leak via {{}} substitution" — documented in mobile-lobe-history.md §Path-discipline pattern table.

### Substitution-domain distinction (amendment #1 codified)

Phase 2 cleanup surfaced a related risk: the brief's original glaze sketch used `"{{baseUrl}}/api/sync/replay"` as the sync queue replay endpoint, ambiguously suggesting compose-time substitution. Verification revealed compose-lobes.mjs regex `/\{\{glaze\.([a-zA-Z0-9_.]+)\}\}/g` matches ONLY tokens prefixed with `glaze.` — plain `{{baseUrl}}` would pass through verbatim, NOT a compose-time token.

Mobile LOBE.md §Glaze contract now explicitly documents the two substitution domains:
- Compose-time: `{{glaze.<dot-path>}}` only
- Runtime (Mobile app environment): `baseUrl`, etc. resolve at app runtime via environment.ts

Glaze stores relative-path-only values (`replay_endpoint_path: "/api/sync/replay"`). Mobile HttpService prepends baseUrl at runtime.
