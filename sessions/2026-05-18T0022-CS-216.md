# CS-216 — ADR-012 Lobe capability profiles + operation invocation

## Summary

Authored **ADR-012** at **v2.31.0** (`fbe75718` on `main`; tag `v2.31.0` =
Pattern-S CI ratification → lobe-regression run `26006996821` event=push
sha=`fbe75718` → **conclusion=success**, all 18 gates). The session began as a
4-source discovery pass (filesystem / bridge / Linear / Boris-Karpathy record),
moved through a /plan that surfaced 4 locked-decision contradictions, then a
**meta-question** (modes vs capabilities) that **overturned the foundation**:
"operating mode" is the wrong Lobe contract primitive. ADR-012 adopts a
**capability-profile primitive** and **scoped-supersedes the mode-framing of
ADR-029/030/031 only** (ADR-027 precedent — bodies, `<MODE>` tokens, shipped
substrates, brownfield ticket fleet all untouched). Doc-scoped: no shipped
substrate, schema, or running code modified. Resolves the CS-208 "ADR-012
skipped" numbering gap (Discovery Contradiction #1).

## The primitive overturn (the load-bearing decision)

Vipin's challenge — *"if auditor lobe has only audit then what is the mode
needed for … mode is a restriction"* — was correct. Three filesystem-grounded
proofs (the Q8 empirical test was decisive):

- **Q8 (shipped-substrate behaviour):** on `change_type=modify` under
  `<MODE>=greenfield-emit`, the builder-tier substrate **defers-with-advisory
  and continues** (`agents/global/backend-lobe-codegen.md:82` step 7e; `:94`
  step 9/Edge 17; ADR-029 §1.D4). Answer = **(d) component-granular
  partial-deferral** — capability-boundary semantics ("this capability is
  unbuilt"), NOT mode-restriction ("forbidden this invocation"). A restriction
  refuses; a capability boundary honestly reports absence and proceeds.
- **CS-215's own text:** *"Each mode has its own input/output schema"* — a
  thing with N independent (input→output) contracts is **N capabilities**, not
  one capability under N restrictions.
- **The tautology:** single-behaviour Lobes (Code Archaeologist) get a mode
  field with one legal value = zero information. `modernize`/`remediate`
  collapse to the same capability bundle differing only by input — overcount.

## Locked decisions (4, operator-ratified after adversarial review)

1. **Model C primitive + explicit deterministic operation selector** —
   free-text intent-inference rejected (Boris regression; the one part of
   Model-C-as-described I pushed back on, ratified).
2. **Closed ADR-owned capability-atom enum** (8 atoms; amendment-governed) —
   open/per-LOBE.md vocab rejected (re-imports drift, no enum to gate).
3. **Capability-keyed Result discriminator**; single-envelope v0.1.0;
   per-capability output-schema split = **named deferred ADR-005/007
   follow-up** (real migration, not silently dropped).
4. **Concept + contract + scoped supersession**; deterministic
   capability-validation gate deferred to its impl ticket (CS-324-class;
   ADR-029 §1.D6 "rides existing machinery" precedent); v0.1.0 enforcement
   is prose, stated honestly in §10.

Earlier /plan Q1 (two-axis) **superseded** by the single capability primitive;
Q3 (no new handoff field) / Q4 (contract + one inline example) held.

## Dogfood walk (Phase D commitment — recorded here, not in the ADR body)

Per the operator-ratified triage-item-2 call (keep dogfood reasoning
surfaced-only; corpus precedent = ADR-029/030/031 carry zero self-application
walks; the load-bearing argument is already in normative ADR §3 generically):

- **Planner / Architect / Test Writer / Doc Writer** = **single-atom
  capability profiles** (e.g. `{produce_plan}`). Under the capability
  primitive this is simply a one-atom profile — **NOT** the zero-information
  tautology it was under the mode framing. (This is CS-215 Axis 4 "pure-build
  Lobes", restated correctly.)
- **Backend Lobe authoring its own future findings-capability** (the CS-215
  Backend-audit POC) runs under `{read_code, write_new_files}` — a normal
  build-equivalent spec→code task. **No bootstrap paradox**: a Lobe building
  the methodology for a capability it does not yet expose is just ordinary
  build work; the capability profile of the *target* Lobe is orthogonal to the
  capability profile of the *building* Lobe.
- **Code-Archaeologist-class** = `{read_code, produce_findings}` — the
  single-purpose profile that exposed the mode tautology in the first place;
  under capabilities it is the contract, with no redundant selector.
- Conclusion: the design holds under self-application. The generic argument
  lives in ADR-012 §3 (extractable); this concrete Colobbo-pipeline walk lives
  here (the corpus-correct surface, read at `/cibo-start-session` Step 1).
