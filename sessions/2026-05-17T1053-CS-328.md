## Keystone design (ADR-031's primary contribution — it AUTHORS, not derives)

- **Selector axis (artifact-determined, not the prompt's guess):** two-level keyed-hint `{ <dialect>: { <component.type>: idiom-or-__skip__ } }`. stack-key = `component.target_type` (dialect; allowlist axis); selector = `component.type` ∈ `glaze.database.supported_component_types` (7-enum, capability-gated + tooling-resolved upstream per ADR-009; codegen never recomputes). Structurally identical to backend's 2-level shape — made explicit for CS-330/future inheritance (operator reinforcement #1).
- **The Ware-specific-token skip pattern (the HIGH /rthink caught):** backend keys its skip off a **Ware-agnostic** enum value (`persistence_layer=='none'`; glaze string decorative) — that is ADR-029's *special case*. DB migration-tier tokens (`drizzle-migration` here / `typeorm-migration` in fintech) are **Ware-specific** → there is no Ware-agnostic enum to key off, so the skip is a **load-bearing parsed `__skip__` sentinel**: `__skip__ <advisory_code> — <text>`, whitespace-split, token[0]==`__skip__`, token[1]=snake_case advisory; malformed → `internal_error` (explicit edge), never silent-emit. ADR-031 §1.D3 authors this as the general case; **CS-330 (Migration codegen) inherits ADR-031 §1.D3, not ADR-029's enum-key special case** (its tokens are also Ware-specific).
- **Fork collapse (operator-confirmed):** prompt FORK C (EAV materialisation) + FORK E (index/constraint) are NOT separate decisions — just keyed-map entries. ADR-009 substitution happens at the Lobe spec stage; codegen consumes post-substitution `component.type` and never recomputes. The 7-enum enumerates the whole design space; the sentinel encodes the v0.1.0 migration boundary.
- **Gate-9 coupling discharge (the CS-328 F-precondition):** Backend↔Database tooling-agreement (ADR-011 / CI Gate 9) discharged **by construction** — codegen consumes the spec-resolved `component.type` (which encodes the tooling decision) and never recomputes. Gate 9 verified green (`drizzle↔drizzle, raw_sql_modern↔raw_sql per ADR-011`).
- **Real-spec dry-run grounding = `lobes/<lobe>-lobe/test-fixtures/`, NOT a pipeline-runs dir** (operator reinforcement #2 — `scripts/.pipeline-runs/` does not exist; canonicalised in ADR-031 §1.D6 for future builder-tier ships, correcting CS-327's COL-231-specific framing).
