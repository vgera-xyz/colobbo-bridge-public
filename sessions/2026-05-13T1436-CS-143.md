## Architectural decisions (codified in ADR-021)

| # | Decision | Rationale |
|---|---|---|
| D1 | Pipeline-stage boundary vs CS-49 | CS-143 reviews specs (Lobe JSONs) pre-PR; CS-49 reviews code on PR-open. Different artifacts, different timing, complementary not redundant |
| D2 | Two-layer L1+L2 with cascade | L1 deterministic (no model calls) covers most issues; L2 ensemble reserved for genuine semantic ambiguity. cascade_strict default true |
| D3 | 8 upstream Lobes pinned per ADR-007 (PlanResult never loop-back-targeted) | Including Migration + Mobile catches cross-cutting concerns; Planner is single-shot |
| D4 | Workspace resolution = per_lobe_aggregator | No upstream Lobe change; reads each Lobe's `glaze.<lobe>.output_dir` per feature_id. shared_root reserved for future Wares |
| D5 | API-contract-shape rule lives at L1 (not L2-Z) | Source-of-truth is internal to Lobe outputs (`ArchResult.interface_contracts` + `BackendResult.endpoint_signature` + `FrontendResult.interface_contracts`) — deterministic keyset comparison, no AI needed |
| D6 | Per-stack type-check commands in Glaze | `tsc --noEmit` insufficient for web-angular + Colobbo-Mobile (strict=false top-level); per-target build commands honour pre-submit-build-checks rule |
| D7 | γ for L2-Z coherence, α for L2-X + L1 local | Coherence requires both sides of disagreement; local findings need only the finding |
| D8 | Single Claude Opus 4.7 default for L2 (pluggable) | CS-49 already runs 3-model ensemble at PR layer; spec-stage doesn't duplicate. Pluggable via Glaze for Wares without CS-49 |
| D9 | Convergence: max 3 iterations + lock monotonicity + Architect-escalate | 3-iteration cap with telemetry; locked Lobes don't oscillate; Architect contradiction = decomposition wrong → escalate |
| D10 | Output contract: 8 machine fields + cost_estimate_cents + audit page | Machine for downstream (Test/Doc/PR-Raiser); human audit at audit_path_template |
| D11 | Pre-flight pending-review check | CS-143 refuses to start if any upstream Lobe has unresolved `human_review.pending_dir` items |
| (ship procedure) | Pattern Q resnap + Pattern R 24h-stability honoured at every Lobe ship | Codified directly in ADR-021 §Ship procedure |
