# CS-340 — Backend Lobe v0.3.0 build+audit (ADR-012 capability-profile POC)

## Summary

First Lobe to exercise the **[[../../decisions/adr-012-lobe-operating-modes|ADR-012]] capability-profile primitive end-to-end on real substrate** — the POC ADR-012 §11 reserved for "each Lobe's own ticket". The handover framed this as CS-215 work; discovery falsified four handover assumptions, the three HIGH ones were put to the operator as forks, and the resolved direction was executed under a **newly-filed CS-340** (parented to **CS-138**, not CS-215 — CS-215 is a Backlog *taxonomy* ticket whose AC is "file 8 children" and whose own text routes capability-extension to the build-Lobe parents; Pattern-G verified CS-138 had zero children). ## Discovery contradictions (handover vs shipped artifacts) → operator fork resolution

1. **CS-215 scope** — CS-215 is a Backlog parent (AC = file 8 children; 1/8 done = CS-216/ADR-012); its text routes Backend audit to a CS-138 child. → Fork **Q1=B**: file the CS-138 child (CS-340), implement under it.
2. **`<MODE>` rename** — handover locked `<MODE>`→`<OPERATION>` in `prompts/codegen.md`; ADR-012 §8/§14 say that token is physically unchanged (framing-only supersession, no fleet-wide rewrite). → Fork **Q2=A**: `<MODE>` untouched.
3. **LOBE vs codegen sub-agent** — handover put audit on `agents/global/backend-lobe-codegen.md` (ADR-029 §2 codegen-only; ADR-012 §8 "shipped sub-agents untouched"). → Fork **Q3=A**: audit lives on the LOBE methodology; sub-agent untouched.
4. **(MED) profile mismodel** — handover's `[read_code, write_new_files, produce_findings]` is wrong (LOBE.md doesn't write files). Shipped `[read_code, produce_plan, produce_findings]`; R1 (open, non-blocking): `produce_plan` for build's spec-production is an ADR-012 §3 interpretation — remedy if rejected is an ADR-012 amendment, never an atom invented here.

Compose-v2-unshipped worry (rthink gap #7) was falsified during discovery — CS-194 shipped 2026-05-07.
