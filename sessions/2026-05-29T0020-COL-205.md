### Phase 4 — COL-186 systemic finding (high-value security audit)

Surfaced during BootsOnGround SP-body reads. The dynamic-SQL injection pattern is **systemic** across SPs ending in `_Paging`, `_Filter`, `_Export`, and many `_GetAll` variants:

- `SP_BootsOnGroundGet`: `EXEC(@sqlStr)` with raw `@filterCriteria` + `@sortby` string-concatenation injection
- `SP_BootsOnGroundGet_Paging`: Two-phase EXEC (INSERT #Table via dynamic + SELECT via dynamic); raw `@sortDataField` + `@sortType` injection
- `SP_BootsOnGroundGet_Filter`: **CRITICAL** — `@columnName` direct-injected into SELECT clause without whitelist

Mitigation strategy documented in wiki COL-186 callout. Worker rewrite MUST translate to safe Drizzle predicates with column allowlists; never preserve raw injection in TypeScript port.

## Substrate / process findings worth noting (per `feedback-state-drift-class` — codify only if recurring)

1. **Python regex catastrophic backtracking on certain SP bodies.** Pre-computing Convention 2 difficulty across all 1000 SP files (literal-strip + multi-line patterns) hung repeatedly; killed multiple background jobs. Mitigation deployed: skip literal-stripping; use simpler line-anchored regex. Even simplified, foreground job didn't complete in ~3 min. **Resolved by deferring per-SP deep-analysis to per-endpoint smoke verification.** The hand-analyzed BootsOnGround set is the reference pattern. Lesson: for large SP bodies with mixed string-literal patterns, regex-based analysis IS unreliable; would need shell tools (grep -c per signal per file) or AST parser.

2. **Resolver coverage gap from delegation aliasing.** 613 of 662 endpoints have controller delegation patterns like `_woRepo.MethodName` (alias field) where the field's concrete type isn't parseable by static regex. Resolved 49 (7.4%) HIGH-confidence; rest LOW-confidence via SP-name-pattern heuristic. **Per-endpoint smoke verification will resolve at invocation time** — worker can read controller body in full and resolve chain.

3. **Side-effect detection undercount.** Email/SMS/notification/queue/cache regex too narrow; returned 0 across resolved subset. Reality almost certainly higher (injected interfaces, factory patterns). **Flag at smoke time per endpoint** — wiki §Detection gaps documents this.

4. **Plan-mode iteration discipline held at 3 rounds + 2 additive rounds** (4-bucket scope addition; 7-angle expansion; auth-framing additive edits). Each addition was strict-fix-modifies-spec per `feedback-concrete-fix-modifies-actual-spec-not-margin-caveat`. No backtracking on prior locks.
