## CS-391 — Pattern-A controller rebake (commit `53c0ff288`)

- **Convention 4** rebaked Pattern B (inline-route handler) → **Pattern A** (route → controller → service): controller at `src/controllers/<x>Controller.ts` (class + singleton), thin delegating route, MOVE+KEEP idioms both Pattern A.
- **Step 4i** gained existing-module **path-discovery** (discover the module's actual controller/service/route locations across api_node's two house styles) + controller append-idempotency; **Step 7** gained a controller-preservation check.
- **Company-scope authz: worker emits NONE.** `recordScopeMiddleware` (`src/permissions/recordScopeMiddleware.ts`) is mounted globally in `src/index.ts:129` (after `authenticateJWT`, before every `/api` route); 0/48 controllers call `assertCompanyAccess` (the one in `riskThresholdRoutes.ts` is a one-off). Convention 3 collapsed into §Project wiring §2 "Auth posture" (documentary, mirrors Convention 5's pointer pattern).
- **Dangling citations fixed:** Convention 4's glaze source (`.migration` namespace) was retired by ADR-037/CS-368; only Convention 2 (ADR-035 §1.D2) remains a verbatim fidelity-baked block.

## CS-390 — MSSQL-MOVE idiom, gate-only (commit `904040fef`)

- **Convention 4 MOVE idiom** is now dialect-aware: SERVICE selects by the endpoint's resolved data-source dialect — MSSQL uses `await getMssqlDb()` (`config/drizzle-mssql.ts`, async getter), predicates from `'drizzle-orm-mssql'`, tables from the **curated** `db/mssql/schema.ts` (`mssqlTable` from `'drizzle-orm-mssql/mssql-core'`). Controller + route layers (Pattern A) are dialect-agnostic, unchanged; no per-controller authz reintroduced (CS-391 invariant preserved — verified).
- **Schema promotion = GATE-ONLY (operator Option 1).** MSSQL MOVE requires the table + needed columns already in the curated `db/mssql/schema.ts`; absent/incomplete → flag `SCHEMA-PROMOTION-NEEDED` (naming table + missing columns) + force KEEP. The worker does NOT auto-promote from the full 397-table introspection dump (`mssql-schema.ts`).
- **Why no auto-promote (verified):** the dump and the curated schema DISAGREE on the same table. `tblWorkOrderType` — dump: `datetime` / `varchar(150)` / no `.notNull()`/`.identity()`; curated (what COL-296 actually queries): `datetime2` / `varchar(500)` / `.notNull()` / `.identity()`. Copying the dump verbatim regresses types; inferring the hand-tuning is schema-design (Boundary 11 forbids). Boundary 11 extended to MSSQL.
