## What was done

**Per Lobe (identical discipline):** `/rthink` before code → `/cibo-new-lobe <name>` scaffold (4-fragment, ungated, Gate-19 clean, prose filled) → hand-derived `output-schema.json` + `test-fixtures/` (cases.json + per-case expected.json + byte-identical glaze-snapshot.json + run.sh) + `scripts/validate-<name>.mjs` (zero-dep) → added `tolerant-reader@1.0.0` include + `glaze.<name>.compatibility.consumed_fields` **from birth (no semver pins anywhere)** → `glaze.<name>` namespace both Wares (byte-identical `compatibility`) → local gates green → one commit → VERSION bump + `registry-regen` → tag → CI-green confirmed.

**Born-tolerant-reader detail:** `consumed_fields` reference only TOP-LEVEL producer schema properties (Gate-2 δ requirement) — Performance: `backend:[components]`, `database:[components, write_targets]`; Design: `arch:[components, interface_contracts]`. Nested fields (persistence_ops, table_definition) are read *inside* those. `error.type` carries **no** `incompatible_*_schema` value in either schema — a MAJOR-bumped upstream is tolerated, not errored.

**DAG wiring (all additive; no MAJOR bump):**
- `glaze.lobes[]` += performance-lobe (v2.54.0) then design-lobe (v2.55.0), both Wares.
- Gate-2 registry (`scripts/verify-lobe-compat.sh`): `performance-backend` + `performance-database`; `design-arch`. **No `design-frontend` entry** — CS-406 discipline (see below).
- Gate-5 coverage loop (`.github/workflows/lobe-regression.yml`) + `verify-gate5-lobe-coverage.sh` (Gate 15) kept in lockstep with `glaze.lobes[]`.
- Code Reviewer `lobes_locked_final` + `lobes_modified_during_review` += `"performance"` then `"design"` — schema **1.2.0 → 1.3.0 → 1.4.0** (two additive MINOR enum-adds). Tolerant-reader consumers (test/docs read `lobes_locked_final` via `consumed_fields.review`) unaffected; deployed `agents/global/code-reviewer-lobe.md` recomposed (Gate 14).
- New per-Lobe CI gates: **Gate 22** (Performance harness) + **Gate 23** (Design harness), mirroring the fleet convention (every live Lobe runs its fixture harness as a dedicated gate).

**CS-406 phantom-coupling discipline (Design):** the new `visual_layout_contract` declares `shape` but **OMITS `consumed_by`** — the CR coherence-check only walks entries with both, so no consumer binding is asserted before a real read exists. Frontend received an **advisory/optional** DesignResult note only (its `interface_contracts` handling stays opaque pass-through); **no `consumed_fields.design` pin, no Gate-2 `design-frontend` entry**. This was the operator's explicit pre-directive: don't create the coupling before the read exists (the CS-406 lesson, applied preventively).
