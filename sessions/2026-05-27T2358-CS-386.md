## Substrate findings worth noting (NOT auto-memory write threshold)

Per `feedback_state_drift_class` — codify only if pattern re-surfaces. Recording here for future-self.

1. **PATCH-vs-MINOR class divergence at the audit-log layer.** M2 was MINOR because the BackendResult schema enum `advisories[].type` is closed → new enum value = schema bump → MINOR. CS-386 was PATCH because the analogous advisory lives in `audit_log.json skipped[].reason` which is documented in markdown prose only — NOT in `lobes/backend-lobe/output-schema.json`. The audit_log is the codegen agent's local output convention, not a downstream-consumed schema. **Worth a one-line auto-memory if it recurs**: codegen-agent audit-log enums are prose; downstream-consumed Lobe-result enums are schema-bound; the bump class differs.
2. **Drive-by gremlin pattern recurrence.** This is the Nth time (definitely 2+, possibly 3+) the `" 2"` macOS-iCloud silent-dup has appeared mid-session and required a `rm`. The auto-memory `feedback_macos_space2_duplicate_corrupts_git_refs` is already canonical. CS-320 macOS gremlin backlog ticket exists for systemic cleanup. No new action; just a tally.
3. **Validator-less S0 step — no automated regression guard.** The S0 step's resolution logic is prose-executed by the LLM at runtime; there's no harness that mechanically exercises "does the agent pick CANDIDATE_A on api_node". This is a known v0.1.0 gap (feature-pipeline §10.2 "validator-less codegen handoff contract"). An end-to-end harness for the codegen agent (feed it a minimal BackendResult + glaze pointing at multiple repo shapes, assert it picks the right candidate or emits the right advisory) is the analogue of `probe-smoke` for the agent side. **Not filing pre-emptively** per `feedback_state_drift_class` — codify only if a 2nd codegen agent (frontend/mobile/database) trips the same path-resolution bug.
