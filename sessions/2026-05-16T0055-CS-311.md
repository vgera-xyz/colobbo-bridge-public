## Learnings (methodology rules — future cycles inherit these)

1. **ADR-numbering discipline.** ADR numbers are reserved by EITHER (a) `wiki/decisions/index.md` registration OR (b) filesystem creation of `adr-NNN-*.md`. Prose mentions in CHANGELOG / bridge entries ("candidate ADR-NNN") are **NOT** first-class reservations. The next builder takes next-free per `cibo-decide` auto-numbering regardless of unbuilt prose reservations elsewhere. Applied here: CS-311 legitimately took ADR-027 (registry never reserved it; no file existed); WP-NEW-4 `--force-with-lease` ADR is re-assigned to **ADR-028** when authored (operator-confirmed). Artifact-rule-wins over prose (`feedback_artifact_rules_over_ticket_prose`).
2. **Pre-ship blocking-verifier sweep must include `scripts/verify-compose-fixtures.sh` (Gate 14) whenever `glaze.lobes[]` changes.** Adding a Lobe to `glaze.lobes[]` requires its composed `agents/global/<name>.md` committed (byte-identity enforced). `verify-glaze-shape.sh` + `verify-gate5-lobe-coverage.sh` + the fixture harness do NOT catch the missing-agent drift. This cost a post-ship CI RED + fix-forward this session.
3. **Stale memory correction.** `project_compose_v2_architect_agent` ("all shipped Lobes ship as LOBE.md only until compose.sh v2 exists; ignore /maintain HIGH flags for missing agent files") is **OUTDATED** — compose.sh v2 / CS-194 shipped; all 10 Lobes now have committed `agents/global/<name>.md`; Gate 14 enforces byte-identity. Future sessions must NOT rely on that memory.
4. **canonical_pages membership** must be checked via `jq '.wiki.canonical_pages[]|select(.path==$p)'`, never ad-hoc `jq | grep` — a false-negative grep misled the R11 reasoning this session (caught only by invariant (k) HARD-FAIL).
