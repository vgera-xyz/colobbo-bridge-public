# CS-338 — Graphify cross-community wiki links blank: root cause + fix

## Summary

Discovery-first investigation + fix of a **100%-universal Layer-1 degradation**: every community/file article under `graphs/*/graphify-out/wiki/` rendered `## Relationships` as "No strong cross-community connections detected" — all repos, all generation dates — while raw `graph.json` held hundreds–thousands of cross-community edges and `GRAPH_REPORT.md` computed the bridges correctly. Root cause **confirmed at source**: graphifyy 0.4.23 `wiki.py:_cross_community_links` reads the in-memory node attr `G.nodes[n]['community']`, but `build_from_json(ast)` (`build.py:56`, pre-clustering extraction) never sets it → guard never passes → empty-state branch always fires. `to_wiki()` already receives the authoritative `communities` dict but that one function ignores it (every other exporter derives via `_node_community_map`). Upstream defect; fixed locally by annotating `G` before `to_wiki()` (mirrors graphifyy's own `export.py:943`). Shipped **v2.32.1**, 18-gate tag-ratified, all 4 graph-bearing repos regenerated + guard-verified. CS-338 → Done.
