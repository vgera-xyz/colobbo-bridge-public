# CS-294 — Regression Test Lobe substrate v0.1.0 ship + first `(aa)` invariant + first capability-primitive wiring

## Summary

Full-session ship of the 13th Lobe in the Motor Lobe pipeline: the **Regression Test Lobe** — fourth multi-upstream consumer per ADR-007 §Generalisation (Migration + Arch + Backend + Review) that produces RegressionResult equivalence findings at migration time. Consumes `MigrationResult.legacy_method_body_summary` (body-read source of truth per ADR-035 §3) and compares against emitted modern code via the L2 equivalence-analysis sub-agent (separate primitive, same release). Substrate-only at v0.1.0; Steps 3/4/5 (Compare/Diagnose/Recommend) stubbed pending Code Reviewer L2 orchestration ship.

Shape ratification preceded the build via a /rthink session that surfaced + closed the four-day CS-294 framing trajectory: 2026-05-16 Option C (Playwright suite promotion) → 2026-05-17 CS-332 brownfield-trust → 2026-05-19 Phase-A prompt → 2026-05-20 system-level reframe. Operator pushed back on my too-quick repin from a hypothetical ArchaeologyResult upstream (the 2026-05-17 CS-332 proposal) to MigrationResult — the answer is that the repin is a **correction**, not a compromise: ADR-035 (shipped 2026-05-20, one day before this session) made MigrationResult.legacy_method_body_summary the canonical structured legacy-behavior spec; the CS-332 ArchaeologyResult proposal predated ADR-035 by 3 days and is a stale prior. Different use cases on different taxonomic axes — CS-215 Code Archaeologist targets module-level synthesis for human KT; this Lobe targets per-target trust verification.

Also discharges ADR-026 bundled obligation as the **FIRST `(aa)` ship** — the single-letter Latin invariant alphabet was exhausted at `(z)` (CS-145 Doc Writer). Forward-format contract `\([a-z]{1,2}\)` documented in `verify-glaze-shape.sh`; `(aa)` block landed for `glaze.test.regression.compatibility` byte-identity; canonical catalogue extended; ADR-026 Status flipped from "Accepted (design + policy only)" to "Implemented at v2.40.0."

## Decisions resolved this session

### 1. Spec shape: Shape B with 4-upstream MigrationResult-pin

Three candidate shapes triangulated from four-day CS-294 framing trajectory. /rthink session ratified Shape B with 4-upstream pin (Migration + Arch + Backend + Review). Options A (CS-103 promotion to forward Playwright suite) + C (system-level cross-PR batch-drift catcher) explicitly rejected with rationale in ADR-036 §Alternatives.

### 2. ArchaeologyResult repin — CORRECTION not compromise

Operator pushed back on my too-quick repin from a hypothetical ArchaeologyResult upstream to MigrationResult. TIER-1 verification: MigrationResult v2.0+ (post-ADR-035, shipped 2026-05-20) carries the full structured legacy-behavior spec (`legacy_method_body_summary` with primary_sp + dependent_sp recursion + dto_field_list + composition arrays + empty_recordset_branch + tsql_procedural_signals + signature + source_ref). The 2026-05-17 CS-332 chat-side comment proposing ArchaeologyResult predated ADR-035 by 3 days — a stale prior. CS-215 Code Archaeologist's actual purpose (module-level synthesis for human KT per CS-215 axis 2) is a different use case on a different taxonomic axis. Pin to MigrationResult IS the architecturally-correct design given the shipped substrate, not a contortion around a missing dependency.

### 3. Steps 3/4/5 stubbed (deferred — not a scoping choice)

Comparison logic (Step 3 Compare / Step 4 Diagnose / Step 5 Recommend) genuinely blocked on Code Reviewer L2 orchestration ship (no pipeline-runner pattern exists yet for spec-stage L2 sub-agent invocation). Substrate ships full contract + schema + L2 sub-agent file; dispatch-ready once upstream lands.

### 4. Substrate Lobe joins `glaze.lobes[]` at v0.1.0

Test Writer + Doc Writer are both in `glaze.lobes[]` despite substrate-only / stubbed-steps shape (the test-writer-lobe LOBE.md prose claiming otherwise is stale). Precedent: substrate-only Lobes register at ship time; Gate 5 portability grep handles via Lobe-extractable methodology. Regression Lobe ships in-line.

### 5. Fixture directory renamed (portability fix mid-build)

Operator's prompt said "api_node-minimal smoke fixture" — but the underscore form `api_node` is in the forbidden-token grep pattern (it's the literal Colobbo stack identifier in repo paths). Test Writer's existing fixture uses hyphen form `api-node-minimal`. Renamed mid-build to match Test Writer convention: `lobes/regression-lobe/test-fixtures/api-node-minimal/`. Updates propagated through cases.json + run.sh + 4 input JSONs + expected/regression-result.json + ADR-036 + L2 sub-agent. All hand-authored substrate files now grep-clean.

### 6. Composed-agent naming convention

`agents/global/regression-lobe.md` (keeps `-lobe` suffix). Matches code-reviewer-lobe precedent; the dominant 8-of-9 pattern drops `-lobe`, but for Lobes whose name conflicts with a generic term (here, "regression" could mean statistical regression vs migration-regression), keeping the suffix disambiguates.
