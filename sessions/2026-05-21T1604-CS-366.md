## First-of-kind: cross-Lobe spec-stage handoff

This is the **first declared cross-Lobe spec-stage handoff** in the agent-system that's NOT mediated through Code Reviewer's `workspace_ref{}` map. Past inter-Lobe data flow was Code Reviewer-mediated (the CS-143 substrate's `workspace_ref{}` is the canonical handoff fabric). Architectural significance:

- **Topology:** Producer (Regression Lobe Step 5) → Consumer (Test Writer Lobe Step 3.6.5) flows VIA filesystem path resolution (`glaze.test.regression.output_dir` + filename glob), NOT VIA Code Reviewer's workspace_ref.
- **Why bypass workspace_ref:** Regression Lobe is downstream of Code Reviewer in the DAG (consumes ReviewResult). At Code Reviewer's runtime, RegressionResult doesn't exist yet. Structural property of the migration-stage pipeline arrangement, not a workaround.
- **CLI flag pattern:** `--regression-input=<path>` is the explicit substitute. Default-resolves via glaze convention; absence behavior varies by intent (advisory for migration, silent skip otherwise).
- **Prototype for future:** the same pattern applies to similar future spec-stage Lobe pairs (e.g. future drift-registry → Regression handoffs when Regression v0.3.0+ ships a `prior_findings` loop-back input).
