# CS-367 filed + migration-worker agent shipped — standalone throwaway agent for COL-205 endpoint migration

## Summary

Session opened with `/cibo-start-session` + a small PR-#246 housekeeping pivot (flip draft→ready + capture inline-JWT ack as a durable PR comment per the operator's earlier brief). ## First-of-kind: throwaway-pattern + source-fidelity-bake discipline

This is the **first throwaway agent** in `agents/global/` that:

- Bakes external-file content verbatim (not paraphrases) — with a build-time fidelity diff that fails the build on drift.
- Uses a Ware-extractable Tier-1 hard refusal gate (the operator's invocation prompt is the override surface).
- Reuses existing Lobes via filesystem-probe + graceful fallback to baked self-check (rather than hard sub-agent dependency).
- Is explicitly throwaway — has a `§Convention re-bake trigger` section signalling it'll be regenerated if source files change OR retired when COL-205 completes.

Architectural significance: shows that not every migration-workflow concern needs to become substrate. A bounded throwaway tool can absorb the per-endpoint disciplines (consumer discovery, classification, conversion, parity, review) without requiring schema changes across the Lobe fleet. This is the operator's `"earned complexity"` principle in action — the substrate rework is deferred until a real recurring need emerges; in the meantime, a single file does the work and dies when the work is done.

The source-fidelity-bake pattern itself (`<<< … verbatim >>>` markers + `diff` against extracted source) is a reusable discipline — any future throwaway or Ware-extractable agent that needs to embed external content can use the same markers + same diff check.
