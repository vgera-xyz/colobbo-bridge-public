## What shipped ‚Äî CS-286 (incidental fix earlier in session)

Commit `3ab5b9c` (2026-05-10): `fix(cs-286): chmod 444 enforced on idempotent branch [CS-286]`. One-line change to `scripts/regenerate-memory-md.sh` ‚Äî added `chmod 444 "$OUTPUT_FILE" 2>/dev/null || true` inside the idempotent branch before `exit 0`. Previously, when MEMORY.md content matched but mode had drifted to 644, the regenerator's no-change branch did not re-assert `chmod 444`. Now it does, unconditionally.

Lessons captured to memory this session:

- `feedback_no_closes_in_commits.md` ‚Äî Never use `closes CS-NNN` / `fixes CS-NNN` in colobbo-agent-system commits; Linear-GitHub auto-close is not wired here; use `[CS-NNN]` bracket convention + manual MCP close. Surfaced when the original commit-message drafted with `closes CS-286` was corrected by Vipin during plan refinement.

## State-drift patterns observed

This session is light on state-drift surfaces. The two operator-correctable patterns surfaced:

- **Pre-flight path discrepancies in user-provided brief.** Three small reference errors in the original CS-283 brief ‚Äî `glaze.json` (actual: `glaze/colobbo.glaze.json`), `rules/bash-jq-reflexes.md` (actual: `rules/shared/bash-jq-reflexes.md`), `status: provenance` (correct per contract: `operational`). All caught during Phase 1 plan-mode exploration via direct filesystem reads. Pattern: verify-by-direct-read of paths cited in briefs before encoding them into plans. Same shape as Pattern #6 (state-drift-patterns will catalogue once CS-284 Phase 4 ships).
- **Mixed precedent on release-vs-bridge commit ordering.** v2.13.0 had release-before-bridge; v2.13.1 had bridge-before-release. Surfaced via `git log` query during plan refinement, not assumed from current memory. Chose v2.13.1 ordering by Vipin's pick; release commit body cites bridge commit hash inline (cleaner narrative pointer than v2.13.0's "tag to follow this commit"). Pattern: verify ordering against precedent before inventing new sequences; if precedent is mixed, surface to operator.

Both patterns are referenceable by CS-284 Phase 4's pattern catalogue page (`wiki/raku-meta/state-drift-patterns.md`) when it ships.
