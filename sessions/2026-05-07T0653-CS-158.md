### Phase A — graphify regression gate
Test subset: 10 files (3 Account* controllers + 3 services + 3 repository interfaces + WebApiConfig.cs). 104 nodes / 134 edges / 8 communities. **Critical finding:** graphify's C# parser is structural-only — does NOT extract attributes / string literals / raw SQL / DI registrations. Regression gate: 3/5 strict-graphify; 5/5 hybrid-graphify+grep. User authorised hybrid approach. Plan reframed Artifacts 4 + 7 as migration-prioritisation signals (NOT security alarm — IPMMasterAPI is decommissioning 2026-07-31).

### Phase B — full graph + cross-repo quality
Cleaned working copy at `/tmp/cs158-source/IPMMasterAPI` excluding: Areas/HelpPage, Scripts (jQuery polluted god nodes — `replace()`/`join()`/`contains()` were top-3 god nodes in first run), Content, Reports, Properties, fonts, Images, UploadedDocuments. Final: 412 .cs files → 3,846 nodes / 5,825 edges / 177 communities. Real god nodes: WorkOrderService (133), WorkOrderController (75), MobileAPIService (74), NotificationService (53), MobileAPIController (47), DispatchWorkOrderService (45), WorkOrderTypeService (37), GlobalService (37), ScheduleService (36), DispatchWorkOrderController (35).

Cross-repo input quality:
- api_node graph: 1,649 nodes / 3,118 edges (5 days fresh; quality OK for TS)
- Mobile graph: 2,840 nodes / 4,948 edges (18 days fresh; zero MSSQL SP refs in source confirmed)
- api_node MSSQL SP usage IS substantive: `apps/api-node/src/shared/storedProcedures.ts` manifest = 79 SPs invoked. Cross-repo SP map is real (NOT near-zero as originally assumed).

Auth surface measurement (Phase B amendment v6): grep across all 74 controllers found **1 `[Authorize]` (commented out) + 0 `[AllowAnonymous]` + 0 global filter + 0 `<authentication>` element in Web.config**. 0 of 660 endpoints have any auth attribute.

## Decision-gate notes (for future-me)

- **Adversarial review pattern:** inline 10-gap review (in lieu of formal /rthink) was the right tradeoff for this session. The discovery-derived gaps (Web API 2 conventions, LEGACY-*.md ground-truth, HelpPage exclusion) were concrete + grounded. /rthink's 7-round methodology would have added rigour but the dilution risk of round-after-round on already-concrete findings was real. User-authorised tradeoff worked cleanly.
- **CS-256 scope inversion** is the highest-leverage architectural decision of the session. Moving SP+table cross-repo work into CS-158 unblocks immediate consumption by CS-140 Migration Lobe; CS-256's reduced scope (auth/middleware/config/EAV) is also more focused than original.
- **Migration-prioritisation reframe** (Artifact 4 + 7 framing) avoided the trap of treating legacy-state findings as security backlog. IPMMasterAPI being decommissioned 2026-07-31 makes those findings migration-design inputs, not patch backlog.
- **Hybrid graphify+grep methodology** is now the canonical pattern for .NET / .NET Framework audits. Document in `wiki/raku-meta/graphify-coverage-audit-colobbo-web.md` Section 1 Bucket-2 + Bucket-3.
