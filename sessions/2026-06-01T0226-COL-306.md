# COL-306 BootsOnGround migration — all 7 endpoints shipped to api_node PR #261

## Summary

Long multi-phase session on `investigation/jignesh-schema-verify`. Started as a read-only schema-verification of Jignesh's PR #368 (Colobbo-Web `MSSQL_SCHEMA/` + `PGSQL_SCHEMA/` live-DB docs), evolved into: (1) **two agent-system edits to `migration-worker.md`** — first an MD-fallback→KEEP schema-lookup tier, then a full **reversal to an author-on-demand model** (worker authors clean colocated Drizzle symbols on demand, guardrail-gated; **CS-390 relaxed**); (2) the **inaugural migration-worker smoke** (DELETE bootsOnGround); (3) **migrating all 7 BootsOnGround endpoints** to typed Drizzle with the COL-186 injection debt fixed in-pass; (4) **the project's first cross-database endpoint** (`getBOGReport` — MSSQL core + native-PG inspection enrichment stitched in Node); (5) opening **api_node PR #261** and updating the COL-306 tracker.

Two agent-system commits to `main` (`f93de82195`, `747e0b688c`). All product code lives in **api_node PR #261** (separate repo). COL-306 (the issue) is at **Dev Done** with PR #261 attached; its description table's 7 BootsOnGround rows now read `PR-raised` + PR #261.

## What was done

### Agent-system (`migration-worker.md`, this repo, main)
- **`f93de82195`** — MD-fallback → KEEP tier: when a MOVE-classified endpoint's Drizzle symbol was absent but the live-DB MD confirmed the table, emit KEEP (call the proven SP) + flag `MD-FALLBACK`, instead of STOP. (Operator Option A.)
- **`747e0b688c`** — **superseded the above** with the **author-on-demand model** (operator reversal): when the symbol is absent, the worker **authors a clean colocated per-module Drizzle symbol** from the live-DB extract and proceeds to a typed MOVE. Gated by a deterministic **G1–G4 guardrail** (native base table, unique name, clean identifier, all columns type-mappable) — junk (FDW foreign tables, sequences, GIS scratch, dups, unmappable types) → KEEP + `MD-FALLBACK`, never a garbage symbol. **CS-390 "Jignesh owns all schemas" relaxed** (worker authors colocated; canonical files still read-only). New flag `SCHEMA-AUTHORED`. Convention-2 verbatim-baked block untouched.

### Product migration (api_node PR #261 — 4 commits: `9fdaa40` schema, `919baa1` 6 endpoints, `1fe586a` cross-DB, `eb20744` flattener fix)
- **Schema verification** of PR #368: confirmed canonical `apps/api-node/src/db/mssql/mssql-schema.ts` (397 tables, on `origin/development`) has `tblBootsOnGround` + `tblBootsOnGroundDeleted`; corrected an early wrong-branch misread (local `col-307` predates the file). MD vs Drizzle agree on names/types/lengths; divergences were nullability/identity completeness only.
- **Inaugural smoke**: ran the migration-worker on DELETE bootsOnGround → typed archive-then-delete in `db.transaction`; caught a real beta-version trap (`drizzle-orm-mssql@1.0.0-beta.22` lacks `.insert().select()` → `tx.execute(sql\`…\`)`). tsc-clean, verified independently.
- **6 MOVEs + COL-186 fix**: `get`/`getFilter`/`export`/`list`/`insert`/`delete` migrated to typed Drizzle with the injection debt fixed — identifier whitelist (`columnName`/`sortDataField` → column symbols, 400 off-list, never `sql.raw`), bound LIKE for value filters, `sortType`→asc/desc whitelist, server-side `pageSize` cap (list 1000; export return-all ≤100k), `totalRecords`/`totalPages` preserved.
- **`.identity()` annotation**: added to `tblBootsOnGround.id` + `tblBootsOnGroundDeleted.deletedId` (PK corrected to `deletedId`) — removed the Row-6 insert cast. Flagged the schema-wide gap (`.identity()` absent on all 376 PKs; generator limitation) as a separate Jignesh pass.
- **First cross-DB endpoint** `getBOGReport`: the legacy PG routine `sp_boots_on_ground_get` joins MSSQL data (FDW foreign tables — `tds_fdw` → prod IPMDB_PD) with native-PG `tblinspection` jsonb. Built as a Node stitch: MSSQL Drizzle core (shared row-2 join graph; LEFT WorkOrderType, no workOrderId, no pagination) + PG Drizzle fetch of `tblinspection.templatejson` + Node flatten to `inspectionData`. New native PG schema `src/modules/inspection/inspection.schema.ts`. Documented in-code as the reference cross-DB pattern.

## Verification notes
- **Tier-1 static shape parity** vs each SP's RETURN/SELECT shape: all MOVEs pass (DELETE + getFilter fully; get/export/list/insert/getBOGReport shape+tsc). Caught + fixed real bugs static-only would miss: dropped `totalRecords`/`totalPages` on the paged endpoints; export 25-row truncation; row-2 missing DISTINCT.
- **Enrichment DATA-verified**: `getBOGReport`'s `inspectionData` flattener checked against **real dev PG data** (`colobbo_db_dev`, native `tblinspection`, read-only) — confirmed structure `array→page.sections[]→items[]→{caption,ans}`, and byte-identical (order-independent) to the routine's own `jsonb_object_agg` across 3 inspections (1821/1590/1527, 107/107 keys). **Caught + fixed a flattener bug** (`eb20744`): items with a missing `ans` serialized away as `undefined` (102 vs 107 keys) → null-coalesced.
