## Keystone design (the reference pattern)

- **Schema-dictated divergence (verified by direct read of `lobes/backend-lobe/output-schema.json`):** a builder Result is produced **pre-Reviewer** → it carries **no `arch_ref`/`review_ref`/`files_to_modify`**. The terminal-tier (test/doc) resolution chain (ADR-024/025 §2.D6) is structurally inapplicable. Builder-tier contract: `--backend-result` + a required **`--arch-result`** (the explicit substitute for the absent `arch_ref`) + `--allowed-stacks` + `--dry-run`; no `--review-result`.
- **F3 resolved by shape, not guard:** `glaze.backend.codegen.framework_specific_hints` is a **two-level map keyed by stack → spec-resolved `persistence_layer`**. Codegen selects the idiom **per-op** from the spec's own `persistence_ops[].persistence_layer` (already resolved migration-state-aware by Backend Lobe Step 5 / ADR-011) — never a blanket hint, never recomputed. The "wrong-idiom" drift class is impossible by construction. Coverage requirement: keyed-map ⊇ the `persistence_layer` enum.
- **CS-216 mode forward-compat:** `<MODE>` parameterized (greenfield-emit wired; brownfield-upgrade/remediate inert). Brownfield change-types → `brownfield_modify_deferred` skip.
- **Pattern-propagation:** the generic pattern is "hints keyed by a per-component selector axis read from the spec"; axis name varies per Lobe (Backend→`persistence_layer`; FE→ui/state axis; DB→dialect/ADR-009). Siblings inherit the keyed-map pattern, not the literal keys.
