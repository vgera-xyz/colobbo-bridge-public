## Substrate finding worth durable memory

**"Structured-emit Layer-0 fragments require deterministic backing — pure LLM-prose execution cannot reliably produce schema-conformant JSON across independent passes."** Three rounds of prose-tightening this session produced three different boundary-condition interpretations (canonical-surface ambiguity → channel-separation + finding-shape ambiguity → would have surfaced timestamp/path-resolution next). The matching ALGORITHM is consistent under prose; the schema-locked JSON EMIT is not. Captured as auto-memory candidate `feedback_structured_emit_needs_deterministic_backing` — written to durable memory ONLY if pattern re-surfaces on Performance / Design / Mobile codegen Lobe ships (per don't-write-on-first-instance threshold). Codified architecturally via [CS-380](https://linear.app/colobbo/issue/CS-380).

**Subsidiary process lesson (also durable-memory candidate `feedback_pre_tag_local_verify_must_be_comprehensive`):** pre-tag local verify must run ALL gate-equivalent scripts, NOT just the Lobe-specific harness for the Lobe whose LOBE.md was edited. M1's 3 tag-attempts cost 3 cycle iterations because pre-tag verify scoped only CR's fixture harness (not Architect's, not wikilinks, not compose-fixtures-drift). A schema bump on Lobe X requires (a) Lobe-X-own-fixture-harness re-run, (b) `verify-wikilinks.sh` (Gate 13), (c) `verify-compose-fixtures.sh` (Gate 14), (d) `verify-glaze-shape.sh` (Gate 1 + invariant-w), (e) `verify-lobe-registry.sh` (Gate 19), (f) `compose.sh deploy --all` (catches stale agent files).
