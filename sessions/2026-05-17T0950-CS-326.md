## Keystone design (the FE selector-axis specifics — ADR-030 §2)

- **Selector axis (verified by direct read of `lobes/frontend-lobe/output-schema.json`):** L1 stack key = `component.target_framework`; L2 selector = `state_handles[].primitive_kind` closed enum (`reactive_value|derived_value|effect`). Two-level map `glaze.frontend.codegen.framework_specific_hints[<target_framework>][<primitive_kind>]`. Codegen inherits the Frontend Lobe's framework-neutral resolved kind per handle (never recomputed) — frontend-neutrality is the FE analogue of backend migration-state-awareness.
- **No `none`-sentinel** — `primitive_kind` is a closed 3-value *required* enum (strictly cleaner coverage obligation than backend's). `state_primitive_unmapped` skip advisory is **pre-declared in `output-schema.json:243`** — the Frontend Lobe's own contract anticipated this codegen skip path.
- **Multi-framework→multi-repo resolution (the FE delta, §2.D3):** one `arch_component_id` fans out to N components (one per `target_framework`); resolution matches `ArchResult.file_mapping[]` by `arch_component_id` **then filters to the entry whose `repo` == `glaze.frontend.framework_repos[target_framework]`**. A literal "mirror the backend single-repo sub-agent" mis-resolves here — called out explicitly in the sub-agent §Methodology.
- **Precondition F2 eliminated by construction (§2.D5):** the framework→repo binding is path-referenced (`framework_repos[<fw>]`→`repos.<key>`, ADR-005); no repo-name literal exists in the sub-agent/template, so the `web_angular`↔`web-angular` stack-key drift is structurally impossible — same eliminate-by-shape discipline ADR-029 §1.D3 applies to F3.
- **Pattern-propagation:** field name `allowed_stacks_at_v0_1_0` is structurally fixed across all three siblings (ADR-029 §1.D6 "mirror exactly"); only the values + the selector-axis name vary per Lobe.
