# v2.23.3 ‚Äî fix force-push regex compound-command spanning

## Summary

Shipped **v2.23.3 PATCH**: a single-line fix to the force-push detection regex in `hooks/global/block-dangerous-commands.py` (line 84). The regex `\bgit\s+push\b.*(\s--force\b|\s-f\b)` ran against the whitespace-normalized **whole** command string; the `.*` spanned `&&`, `||`, `;`, `|`, so a legitimate compound where `--force` belonged to a *different* subcommand (e.g. `git push origin main || { git fetch origin main --force; ‚Ä¶ }`) was blocked pre-execution ‚Äî also killing an earlier `git commit` in the chain. Changed `.*` ‚Üí `[^&|;\n]*?` (negated class cannot cross a command separator). A genuine single force-push has no separator between `push` and its own flag, so real `git push --force`/`-f` (including embedded in a compound) **still blocks** ‚Äî zero force-push false-negative introduced. **NO TICKET SPAWN**; this is **NOT CS-316 scope** ‚Äî `linked_issue: CS-316` is traceability-thread continuity only (mirrors the v2.23.1/v2.23.2 bridge precedent). **CS-316 stays Backlog** (9 of 13 items unchanged; Step-3a auto-transition deliberately overridden ‚Äî see below). ## The fix

```
- if re.search(r"\bgit\s+push\b.*(\s--force\b|\s-f\b)", cmd):
+ if re.search(r"\bgit\s+push\b[^&|;\n]*?(\s--force\b|\s-f\b)", cmd):
```

Line 84 only. Lines 75 (normalization), 78 (push-origin-main guard), 88‚Äì94 (`is_rm_rf`), 97 (DROP TABLE), 102‚Äì104 (DELETE FROM/WHERE) **byte-unchanged**. `--force-with-lease` deliberately still BLOCKED (operator-approved: pure bug fix, no policy widening; the `\b` after `force` already catches it).
