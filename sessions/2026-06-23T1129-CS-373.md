# CS-373 M8 increment-2 — Frontend tolerant-reader migration + Layer-0 fragment (tag v2.49.0)

## Summary

Shipped **M8 increment-2** — the first *real* per-Lobe step of the CS-373 consumer-side tolerant-reader migration (derivation §7, lowest-coupling-first order: **Frontend** is the 1-pin head). Two deliverables, both landed under tag **v2.49.0** (lobe-regression run 28022700854 — **all 21 gates green**, 1m31s):

1. **`lobes/_shared/tolerant-reader/1.0.0.md`** — the new Layer-0 shared fragment, implementing the operator-decided **Fork B option (a)**: one reusable fragment that all 9 consumers inherit via `includes:`, rather than 9 bespoke LOBE.md rewrites. Parameterised (`scope`, `upstreams`, `consumed_fields_ref`); single- AND multi-upstream ready; generic prose (Result-type names arrive via params) so Gate-5 clean.
2. **Frontend Lobe migrated** (`frontend-lobe` 0.1.2 → **0.2.0**) from the semver-range pin to a `consumed_fields[]` declaration.

This is **MINOR** (new substrate fragment + first consumer behaviour change), distinct from M8 increment-1's gate-only PATCH (v2.48.2).

## What was done

**Layer-0 fragment** — replaces a consumer's runtime `schema_version` version-gate with "read the declared consumed fields; tolerate any upstream version." Explicitly preserves the two orthogonal Step-1 checks (payload-parse + upstream-`status` propagation). The runtime remove/rename protection it drops is exactly what the build-time Gate 2 invariant (δ) now provides (derivation §11 Test 2 — *removes VERSION-coupling, not ORDERING-coupling*). `_registry.json.shared_fragments[]` now lists 7 fragments.

**Frontend migration:**
- `glaze.frontend.compatibility` flips `consumes_arch_schema: ">=1.0.0 <2.0.0"` → `consumed_fields: { arch: ["components", "interface_contracts"] }` in **both** glazes (byte-identical per glaze-shape invariant (f) + Gate 2 (δ) parity).
- `LOBE.md` wires `includes: tolerant-reader@1.0.0` (param_hash `a00fcd666d56`), splices the marker into Step 1, removes the version-gate (former Step 1.3) + all 4 `consumes_arch_schema` compose-token sites.
- Fixture `schema-incompat-arch-result` → renamed **`tolerant-reader-future-arch-schema`**; expected outcome flips **error → ready** (a future ArchResult `schema_version 2.0.0` is now tolerated + expanded to 2 components). Assertions + README rewritten; `cases.json` updated.
- Recompose (`agents/global/frontend.md` only changed) + `registry-regen` (core 0.2.0, composite_signature updated, system_version 2.49.0).

## The consumed_fields convention (operator-decided — sets the precedent for the next 8 Lobes)

The derivation/M8-inc-1 one-liner said `consumed_fields.arch = [components, file_mapping, interface_contracts]`. **/rthink falsified it against the artifact**: the Frontend LOBE.md does NOT read `file_mapping` (repos come from `glaze.frontend.framework_repos`, not ArchResult); it reads `components` + `interface_contracts` (domain) plus the `status`/`feature_id`/`intent` handoff envelope. Surfaced as a decision; **operator chose `["components", "interface_contracts"]`** — the convention is now: **declare the DOMAIN fields you read; the ADR-007 handoff envelope is NOT declared** (it is the stable, never-per-edge-removed contract). This applies to Mobile → Architect → Backend → Database → Test → Docs → Security → Code Reviewer.

## Two deliberate non-actions (cascade avoidance)

- **Frontend OUTPUT schema NOT bumped.** `incompatible_arch_schema` is kept in the `error.type` enum as a now-unreachable value (annotated via `$comment`). Removing it = enum-removal = MAJOR output-schema bump → `schema_version` 2.x → outside downstream consumers' `consumes_frontend_schema: <2.0.0` pins (e.g. Code Reviewer, which migrates LAST) → a Gate 2 (γ) cascade RED. Tolerant-reader is consumer-side; it must not touch the consumer's own output contract.
- **Changelog `{{…consumes_arch_schema}}` mention de-braced.** A documentary mention of the removed token in the LOBE.md changelog would have been matched by compose's `TOKEN_RE` (it does not respect markdown code-spans) → unresolved → compose exit 10 → Gate 14 RED. Caught by /rthink's "zero surviving tokens" check; rewritten brace-free.
