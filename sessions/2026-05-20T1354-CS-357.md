## CS-318 validation (the 3-round empirical journey)

**Round 1** (the v0.1 substrate first-pass, pre-redesign session): Code Reviewer halted at `escalated_to_architect` with 3 critical/high architect-domain findings (DTO parity gap, empty-recordset semantic inversion, file-decomposition gap) + 5 HIGH/MEDIUM/LOW (auth gap, route unregistered, SP-registry bypass, file-org divergence, console.error). Root cause: name-based pre-classification + no body-read.

**Round 2** (v2.0 substrate first pass): 7 of 8 Round-1 findings RESOLVED. But 1 new HIGH surfaced — the **same SP-name failure class recursed ONE LEVEL DEEPER**: codegen body-read the PRIMARY SP correctly (`SP_WorkOrderTypeGet` vs the stale catalog's `SP_WorkOrderTypeGetByStatus`) but did NOT recursively body-read the DEPENDENT service methods. 5 of 8 dependent SP names guessed correctly by lucky string-mangle, 3 wrong: `SP_WorkOrderTypeSkillGet` → actual `SP_WorkOrderTypeSkillSetDetailsGet`; `SP_WorkOrderTypeMilestoneGet` → actual `SP_MilestoneModuleLinkGet` (different params); `SP_MenuGet` → actual `SP_GetMenuDetails`. Production runtime crash with "Could not find stored procedure" errors.

**Round 3** (operator-authorized extension of the 2-round bound on scoping evidence — read each of the 8 dependent method bodies; verified CONTAINED: all 8 are flat depth-1 single-SP shapes, no nested compositions). Built: LOBE.md Step 5 dependent-SP recursive body-read (depth-cap 3); output-schema.json `dependent_sp` optional field on all 3 entry types in `legacy_method_body_summary`; codegen consumes `dependent_sp.name` verbatim. Re-emitted: deleted Round-2 service.ts (wrong SP names) + re-ran codegen Round-3 LIVE → greenfield-create service.ts with 9/9 verbatim SP names + correct params; route file idempotent-skip (Round-2 modify-append-handler intact); index.ts idempotent-skip (already mounted). **`tsc --noEmit` PASS exit 0.** **Code Reviewer Round 3: `status: ready`, 0 critical / 0 high / 1 LOW** (doc-comment count drift "6 dependent SPs" — actual 8; chore-followup, non-blocking).

The 2-round bound was extended on **evidence** (scope verdict before build), not blindly. Round 3 closed a class-level substrate gap, not a one-shot ticket bug.
