# v2.23.4 — close WP-NEW-3 DELETE/WHERE compound-spanning false-negative

## Summary

Shipped **v2.23.4 PATCH**: closed the destructive-SQL **false-negative** the
v2.23.3 ship surfaced as bridge **WP-NEW-3**. The `DELETE FROM … WHERE` guard
in `hooks/global/block-dangerous-commands.py` checked the `DELETE FROM` token
and the `WHERE` token with **two independent `re.search` calls over the whole
whitespace-normalized command string**. A genuinely WHERE-less `DELETE FROM`
was **allowed** whenever *any unrelated* compound segment (`&&`, `||`, `;`,
`|`) contained the word `WHERE` (e.g. `psql -c "DELETE FROM t"; psql -c
"SELECT 1 WHERE x"` slipped through). Restructured the block to split the
normalized command on `[&|;\n]+` and require `WHERE` in the **same segment**
as the `DELETE FROM`. Separator class `[&|;\n]` is **byte-identical** to the
v2.23.3 line-84 force-push fix (cross-line consistency; `\n` is dead
post-line-75 normalization but kept for symmetry, exactly as line 84 does).
Bundled an **additive** correction of an inaccurate file-placement clause in
the [2.23.3] CHANGELOG entry. **NO TICKET SPAWN**; **NOT CS-316 scope** —
`linked_issue: CS-316` is traceability-thread continuity only (mirrors the
v2.23.1/.2/.3 bridge precedent). **CS-316 stays Backlog** (Step-3a
auto-transition deliberately overridden — see below).

The session brief prescribed Item 1 as the *same mechanical `.*` →

## The fix

```
- # Block: DELETE FROM without WHERE (case-insensitive)
- if re.search(r"\bDELETE\s+FROM\b", cmd, re.IGNORECASE):
-     if not re.search(r"\bWHERE\b", cmd, re.IGNORECASE):
-         print("BLOCKED: DELETE FROM without WHERE clause is not allowed.", file=sys.stderr)
-         sys.exit(2)
+ # Block: DELETE FROM without WHERE - per command segment.
+ # WHERE must be in the SAME segment as the DELETE FROM; a WHERE in an
+ # unrelated compound segment (&& || ; |) does NOT make the DELETE safe
+ # (WP-NEW-3). Separator class matches the line-84 force-push fix.
+ for _seg in re.split(r"[&|;\n]+", cmd):
+     if re.search(r"\bDELETE\s+FROM\b", _seg, re.IGNORECASE) and not re.search(r"\bWHERE\b", _seg, re.IGNORECASE):
+         print("BLOCKED: DELETE FROM without WHERE clause is not allowed.", file=sys.stderr)
+         sys.exit(2)
```

Single hunk (`git diff`: +6/-3). Lines 1–100 (incl. line-84 force-push regex,
line-97 DROP TABLE, `is_rm_rf`) and 106+ **byte-unchanged** (asserted at apply
time: `count(OLD)==1` pre, `count(OLD)==0`/`count(NEW)==1` post, explicit
line-84 invariant assertion). `BLOCKED:` message string preserved
byte-identical. Edit applied via `python3` heredoc + temp-file + `os.replace`
(Edit/Write deny-listed for `hooks/**`).
