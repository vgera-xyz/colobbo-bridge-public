## Methodology data points (two captured this session)

**Pattern A — two-tool convergence with refinement.** Two independent reads (this thread + R2.54) converged on "diagnose first, original premise may be gone." Diagnostics partially confirmed reframe (PR #78 closed `start-session`/`end-session` raku-brain rename) AND partially rejected it (`test`, `code-reviewer`, 4 product-repo collisions remain). Same shape as state-drift pattern #6 (2026-05-09 daily log) but with refinement: **the second-tool reframe is itself a hypothesis to test, not a conclusion to apply.** Without verification, accepting R2.54's reframe at face value would have FREEZE'd in latent shadowing.

**Pattern B — verdict-shape discipline.** First-pass D2 verdict allowed `undefined — needs test` as an outcome. User flagged: "that's a gap, not a verdict." Refined to require terminal falsifiable verdict (path i / ii-a / ii-b) before exiting D2. Generalises: any diagnostic gating a FREEZE/PROCEED decision must close on a falsifiable answer; gaps deferred for "later test" defeat the purpose of the diagnostic.

**Pattern C — audit-state vs deploy-state divergence (surfaced in correction).** D6 initially flagged web-angular `lint-angular-file.py` as orphan based on filesystem grep. Post-handoff `bash deploy.sh` proved the file IS deploy.sh-managed; the audit captured a transient pre-deploy state (`.claude/` is gitignored, file gets cleaned and re-written on each deploy). Generalises: read-only audits over deploy.sh-managed targets must either (a) run deploy.sh first, OR (b) explicitly distinguish "current filesystem state" from "deploy.sh-contracted state." Conflating the two surfaces false orphan findings. Same shape as state-drift pattern class (convention referenced verbally, not codified mechanically) — the deploy.sh contract IS codified, but a read-only audit that doesn't replay the contract sees stale state.
