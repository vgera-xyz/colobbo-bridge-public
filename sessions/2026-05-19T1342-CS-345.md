# CS-345 — Pipeline Orchestration Primitive

## Summary

Shipped **CS-345** as `v2.35.0` (commits `e912a3882` + `c2998b20a`, tag
`v2.35.0` → `c2998b20a`, 18-gate Lobe Regression Suite **green**). The brief
framed this as "build the orchestrator"; investigation **inverted** that: the
orchestration protocol already exists, is ADR-ratified (**ADR-028**, CS-146),
and was run end-to-end on COL-231. The real gap was (a) no repeatable skill
encoding the protocol (it was hand-followed per run) and (b) no structured
run-record (the COL-231 golden dir has per-stage files but no `run.json`).
CS-345 is the **pay-once operationalization** of the build sub-chain — a
project skill, not a Lobe (ADR-016 forbids a separate Orchestrator; ADR-028
Decision 1 = CC-as-runtime). MVP scope (static, deterministic) unblocks
CS-330 / CS-318 / CS-322.

## Detour — Gate 19 (resolved, lesson recorded)

First `v2.35.0` tag-run failed **Gate 19** (ADR-032 registry drift). Root
cause: `compose.sh registry-regen` stamps `VERSION` into
`_registry.json.system_version`; the VERSION bump alone left it at 2.34.0.
Sole drift = that one line (not the fixtures, not a pre-existing red). Fixed
forward: regen-commit (`c2998b20a`) + tag delete/re-create at the fix commit
(kept clear of the `--force` regex). New memory written:
`feedback_version_bump_requires_registry_regen` — bake "VERSION bump ⇒
`compose.sh registry-regen` + commit before tag" into every ship checklist
that touches VERSION. State-drift class: Gate 19 enforces at tag-time but it
was missing from the author-time pre-tag checklist.
